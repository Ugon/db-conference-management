Procedury
-Dodawanie nowego klienta DONE
-Dodawanie nowej firmy DONE
-Dodawanie uczestnika warsztatu/konferencji  DONE
-Dodawanie konferencji DONE
-Dodawanie nowego progu cenowego DONE
-Dodawanie warsztatu DONE
-Dodawanie dnia (w sensie konferencji) DONE
-Dodanie zapłaty za rezerwację
-Tworzenie rezerwacji (zarówno na dzień jak i warsztat) DONE

Triggery
-Sprawdzanie czy klient nie zapłacił już całości(przy zapłacie za warsztat/dzień konferencji) DONE - constraint paid <= price
-Sprawdzanie ilości wolnych miejsc przy rezerwacji DONE - obliczanie slotsfilled + constraint slotsfilled <= capacity
-Sprawdzanie, czy klient nie rezerwował już warsztatu o danej porze (przy rezerwacji na warsztat) DONE
-Aktualizacja danych dotyczących płatności przy zmianie rezerwacji DONE - zmian byc nie moze
-Aktualizacja dancych odnośnie ilości miejsc w przypadku anulowania rezerwacji/dodawania nowej rezerwacji DONE
-automatyczne wypelnianie czasu reserwacji przy dodawaniu reserwacji DONE na defaulcie
-automatyczne liczenie wydanego hajsu przez klienta

Widoki
-Uczestnicy wybranej konferencji
-Uczestnicy wybranego warsztatu
-Nie opłacone warsztaty
-Nie opłacone dni konferencji
-Wydarzenia, które nie mają uzupełnionych danych odnośnie uczestników
-Podsumowanie dotyczące zobowiązań klienta/klientów (co zarezerwował, ile miejsc, ile do zapłaty, ile zapłacił)
-Nadchodzące wydarzenia (konferencje/warsztaty z datą przyszłą)
-Warsztaty cieszące się największą popularnością
-Podsumowanie o klientach, którzy wydają najwięcej
